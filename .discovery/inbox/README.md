# Inbox - 生の悩み・違和感・思考ログ

このディレクトリは、**人間（human）** が生の悩み・違和感・思考ログを記述する場所です。

PDL がこの内容を起点に、問題を構造化し、仮説を立て、PdM が判断できる形に変換します。

## 使い方

### 1. 新しい悩みを書く

このディレクトリに、新しい Markdown ファイルを作成してください。

**ファイル名の例**:

- `2025-01-15_日報入力の使いにくさ.md`
- `2025-01-15_フォローアップ機能の改善.md`
- `2025-01-15_週次振り返りの効率化.md`

日付プレフィックス（`YYYY-MM-DD_`）を付けると、時系列で管理しやすくなります。

### 2. 何を書けばいいか

以下のような内容を自由に書いてください：

- **悩み**: 「○○がうまくいかない」「○○が不便だ」
- **違和感**: 「○○がおかしい」「○○が不自然だ」
- **思考ログ**: 「○○について考えている」「○○を改善したい」
- **観察**: 「ユーザーが○○で困っている」「○○の操作が複雑だ」

**重要**: 解決策や機能のアイデアは書かなくても大丈夫です。問題や違和感をそのまま書いてください。PDL が問題を構造化し、仮説を立てます。

### 3. テンプレート（参考）

以下のテンプレートを参考に、自由に記述してください：

```markdown
# [悩みのタイトル]

## 何が問題か

[問題や違和感を自由に記述]

## 現状

[現在の状態や状況]

## 理想

[理想的な状態や状況（書けなくても大丈夫）]

## その他のメモ

[補足情報や関連する観察など]
```

## 例

### 例1: 日報入力の使いにくさ

```markdown
# 日報入力の使いにくさ

## 何が問題か

日報を入力する際、毎回同じような内容を書くのが面倒だ。
特に、毎日のルーティンワークを記録するのが大変。

## 現状

- 毎日、同じような内容を手入力している
- テンプレート機能がない
- 過去の日報を参照しながら書いている

## 理想

- よく使う内容をテンプレートとして保存できる
- 過去の日報から内容をコピーできる
- 入力がもっと楽になる
```

### 例2: フォローアップ機能の改善

```markdown
# フォローアップ機能の改善

## 何が問題か

フォローアップを管理する際、どのフォローアップが完了したか、まだ未完了かが分かりにくい。
特に、複数のフォローアップがある場合、優先順位が分からない。

## 現状

- フォローアップの一覧を見ても、状態が分かりにくい
- 完了したフォローアップと未完了のフォローアップが混在している
- 優先順位が分からない

## 理想

- フォローアップの状態が一目で分かる
- 優先順位が明確になる
- 完了したフォローアップを簡単に確認できる
```

## 次のステップ

1. 悩みを書いたら、PDL に知らせてください
2. PDL が問題を構造化し、仮説を立て、PdM が判断できる形に変換します
3. PdM が判断し、Adopt / Defer / Reject のいずれかを決定します

## 注意事項

- **解決策を書く必要はありません**: 問題や違和感をそのまま書いてください
- **機能のアイデアは不要です**: PDL が問題を構造化し、仮説を立てます
- **自由に書いてください**: 形式にこだわらず、思いついたことをそのまま書いてください
