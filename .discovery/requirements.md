# Requirements

このファイルは、PdM が判断可能な粒度の要求を定義します。

## 要求の形式

各要求は以下の形式で記述されます：

```markdown
## R-01: [要求のタイトル]

**対応する問題**: P-01

**要求の説明**: [実装方法やUI詳細には踏み込まず、「何を実現すべきか」を記述]

**受け入れ基準**:

- [ ] 基準1
- [ ] 基準2

**備考**: [補足情報]
```

## 原則

- 各要求は対応する問題（P-xx）を明記する
- 実装方法・UI詳細には踏み込まない
- 要求はすべて特定の問題に紐づいている

---

## R-01: 取り組みログ記録機能

**対応する問題**: P-01（過去の取り組みと思考を時系列で振り返れない）

**要求の説明**:

フォローアップ（エピソード/アクション）を記録する際に、その時の思考や判断、気づきをログとして記録できる機能。ログは時系列で参照でき、「あのときこんなことを思って、こんな取り組みをしたな」という文脈を保存できる。

**受け入れ基準**:

- [ ] フォローアップ記録時に、思考や判断をテキストで記録できる
- [ ] ログは時系列で一覧表示できる
- [ ] ログを後から参照し、編集できる
- [ ] ログにはタイムスタンプが自動的に記録される
- [ ] ログの記録は任意（必須ではない）

**備考**:

- ログの入力負担を軽減するため、簡潔に記録できる仕組みが望ましい
- ログの記録は任意とし、ユーザーが必要と感じたときのみ記録できるようにする
- ログの文字数制限は柔軟に設定する（例: 500文字程度）

---

## R-02: 半期ごとの振り返り可視化機能

**対応する問題**: P-02（半期ごとの振り返りで、目標達成のためにやったことを可視化できない）

**要求の説明**:

半期や四半期といった長期スパンで、目標達成のために行った取り組み（日報、フォローアップ、週次フォーカス等）を目標ごとに集約し、一元的に可視化する機能。どの目標に対して何をやってきたかを俯瞰でき、成果と取り組みの関係性を明確にする。

**受け入れ基準**:

- [ ] 半期や四半期といった期間を指定して、振り返りを実施できる
- [ ] 目標ごとに、関連する取り組み（日報、フォローアップ、週次フォーカス）を集約表示できる
- [ ] 成果（目標達成度）と取り組みの関係性を可視化できる
- [ ] 振り返り結果をエクスポート（PDF、Markdown等）できる（任意）
- [ ] 振り返り結果を次の目標設定に活かせる形で保存できる

**備考**:

- 期間は柔軟に設定できる（半期、四半期、カスタム期間）
- 目標との関連付けは、既存の関連付け機能（BL-03）を前提とする
- 振り返り結果の保存形式は、ユーザーの使いやすさを考慮する

---

## R-03: タイムライン俯瞰機能

**対応する問題**: P-03（全体を時系列で俯瞰する仕組みがない）

**要求の説明**:

目標、日報、フォローアップ、週次フォーカスといった各種データを、時系列で統合して表示する機能。タイムライン形式で全体を俯瞰でき、ある期間の活動や変化を一目で把握できる。データ間の時系列的な関係性を可視化する。

**受け入れ基準**:

- [ ] 各種データ（目標、日報、フォローアップ、週次フォーカス）を時系列で統合表示できる
- [ ] タイムライン形式で全体を俯瞰できる
- [ ] 期間を指定して、特定期間のタイムラインを表示できる
- [ ] データ間の時系列的な関係性が可視化される（例: 目標設定 → フォローアップ → 日報）
- [ ] タイムラインのフィルタリング機能（目標別、データ種別等）が提供される

**備考**:

- タイムラインの表示形式は、ユーザーの使いやすさを考慮する（例: カレンダー形式、リスト形式）
- データ量が多い場合のパフォーマンスを考慮する
- タイムラインの表示項目は、ユーザーがカスタマイズできる（任意）
