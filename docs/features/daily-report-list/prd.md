# 日報一覧・詳細表示機能 要件定義書（PRD）

## 1. 概要
ユーザーが過去に入力した日報を一覧で確認し、任意の日報の詳細を閲覧できる機能。  
振り返りや自己分析の起点となる。

## 2. ユーザーストーリー
- 新卒社員として、自分が過去に書いた日報を一覧で確認したい
- 特定の日の日報を選んで詳細内容を見たい
- 最近の日報から順に並んでいてほしい

## 3. 機能要件

### 3.1 一覧画面
| 項目 | 説明 |
|:-----|:-----|
| 表示内容 | 日付、できごと（冒頭N文字）、よかったこと有無アイコン等 |
| 並び順 | 新しい日付順（降順）をデフォルト |
| ページング | 必要に応じて無限スクロール or ページネーション |

### 3.2 詳細画面
| 項目 | 説明 |
|:-----|:-----|
| 表示内容 | 日付、できごと、学び、よかったこと、改善点/アクション |
| 編集導線 | 「編集」ボタンで入力画面へ遷移（Phase 1では編集機能は任意） |

### 3.3 操作フロー
1. 一覧画面を開く
2. 日報カード or 行をタップ/クリック
3. 詳細画面へ遷移し、全項目を閲覧

## 4. 非機能要件
- モバイル/PC両対応（レスポンシブ）
- 一覧の初期表示は高速（ローディング体験に配慮）

## 5. 成功基準
- ユーザーが過去の日報を迷わず探して閲覧できる
- 一覧〜詳細の遷移がスムーズ
