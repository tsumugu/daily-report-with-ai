# 日報一覧・詳細表示機能 UI/UX設計書（UI Design）

## 1. 概要
日報一覧画面・詳細画面のUI/UX設計方針を定義する。

---

## 2. 一覧画面

### 画面構成
- ヘッダー：タイトル「日報一覧」、新規作成ボタン
- リスト部：日報カード形式で縦並び
- フッター：ページング or 無限スクロール

### UIコンポーネント
| 項目 | コンポーネント | 備考 |
|:-----|:--------------|:-----|
| 日報カード | Card | 日付、できごと（冒頭50文字）、アイコン表示 |
| よかったこと有りアイコン | Icon (✨) | GoodPointが1件以上ある場合に表示 |
| 改善点有りアイコン | Icon (📝) | Improvementが1件以上ある場合に表示 |
| 新規作成 | Button (Primary) | 「日報を書く」 |
| ローディング | Spinner | 追加読込時に表示 |

### ワイヤーフレーム（イメージ）

```
┌──────────────────────────┐
│  日報一覧     [＋日報を書く] │
├──────────────────────────┤
│ ┌────────────────────┐   │
│ │ 📅 2025-12-04      │   │
│ │ 今日はミーティングが... │
│ │ ✨ 📝              │   │
│ └────────────────────┘   │
│ ┌────────────────────┐   │
│ │ 📅 2025-12-03      │   │
│ │ 午前中にコードレビュー... │
│ │ ✨                 │   │
│ └────────────────────┘   │
│ ...                      │
│      [読み込み中...]      │
└──────────────────────────┘
```

---

## 3. 詳細画面

### 画面構成
- ヘッダー：戻るボタン、日付表示
- コンテンツ部：各項目（できごと・学び・よかったこと・改善点）

### UIコンポーネント
| 項目 | コンポーネント | 備考 |
|:-----|:--------------|:-----|
| 各セクション | Card / Section | ラベル＋本文 |
| 編集ボタン | Button (Secondary) | 「編集する」（Phase 1では任意） |

### ワイヤーフレーム（イメージ）

```
┌──────────────────────────┐
│  ← 2025-12-04    [編集]  │
├──────────────────────────┤
│ ■ できごと               │
│ ┌────────────────────┐   │
│ │ 今日はミーティングが   │
│ │ 3件あり、午後は...    │
│ └────────────────────┘   │
├──────────────────────────┤
│ ■ 学び                   │
│ ┌────────────────────┐   │
│ │ ファシリテーションの   │
│ │ コツを学んだ...       │
│ └────────────────────┘   │
├──────────────────────────┤
│ ■ よかったこと ✨        │
│ ┌────────────────────┐   │
│ │ ・〇〇がうまくいった   │
│ │   要因: △△を意識した │
│ └────────────────────┘   │
├──────────────────────────┤
│ ■ 改善点/アクション 📝   │
│ ┌────────────────────┐   │
│ │ ・時間管理を改善したい │
│ │   アクション: 朝に...  │
│ └────────────────────┘   │
└──────────────────────────┘
```

---

## 4. レイアウト方針
- モバイルファースト
- カードUIで視認性を確保
- PC時は中央寄せ＆最大幅制限

## 5. インタラクション
- 一覧カードタップで詳細へ遷移
- 無限スクロール or 「もっと見る」でページング
- 詳細画面から編集画面への導線（編集機能実装時）

## 6. デザイントークン
- Shadow-CN / Tailwind CSSベース
- デザインシステム準拠

## 7. 状態別表示

### 一覧画面
| 状態 | 表示 |
|:-----|:-----|
| 初期ローディング | 画面中央にスピナー |
| 追加読込中 | リスト下部にスピナー |
| データなし | 「まだ日報がありません」+ 新規作成ボタン |
| エラー | 「読み込みに失敗しました」+ 再読込ボタン |

### 詳細画面
| 状態 | 表示 |
|:-----|:-----|
| ローディング | 画面中央にスピナー |
| 項目が空 | 該当セクションは非表示 or 「（なし）」表示 |
| エラー | 「読み込みに失敗しました」+ 戻るボタン |
