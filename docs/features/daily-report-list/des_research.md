# 日報一覧・詳細表示機能 デザインリサーチ（Des Research）

## 1. PRD要求サマリ
- 一覧画面：日付、できごと（冒頭N文字）、よかったこと有無アイコン等
- 並び順：新しい日付順（降順）
- ページング：無限スクロール or ページネーション
- 詳細画面：全項目閲覧、編集導線
- モバイル/PC両対応

---

## 2. 一般的なUI/UXパターン

### 2.1 一覧画面のパターン

| パターン | 特徴 | 適用ケース |
|:---------|:-----|:-----------|
| **カードリスト** | 情報量多い、視認性高い | 日報、ブログ記事 |
| **テーブル** | 比較しやすい、情報密度高 | 管理画面、データ一覧 |
| **タイムライン** | 時系列が明確 | SNS、活動履歴 |

### 2.2 一覧項目の表示

| 要素 | ベストプラクティス |
|:-----|:------------------|
| **日付** | 目立つ位置、相対表示（今日、昨日）も有効 |
| **概要** | 冒頭50〜100文字、省略記号（...） |
| **アイコン** | よかったこと有無、改善点有無をアイコンで表示 |
| **タップ領域** | カード全体をタップ可能に |

### 2.3 詳細画面のパターン

| 要素 | ベストプラクティス |
|:-----|:------------------|
| **ヘッダー** | 日付表示、戻るボタン |
| **セクション分け** | できごと / 学び / よかったこと / 改善点 |
| **編集導線** | 右上 or 下部に編集ボタン |
| **スクロール** | 縦スクロールで全項目閲覧 |

## 3. 類似サービスのデザイン事例

| サービス | 一覧デザイン | 特徴 |
|:---------|:-------------|:-----|
| **Notion** | カード + リスト切替 | 柔軟なビュー |
| **Apple Notes** | シンプルリスト | 日付 + 冒頭テキスト |
| **Evernote** | カード | サムネイル + 概要 |
| **nanoty** | テーブル形式 | 業務向け、情報密度重視 |

## 4. 推奨デザイン方針（本プロジェクト向け）

### 一覧画面

#### レイアウト
- モバイル：カードリスト（縦1列）
- PC：カードリスト or 2列グリッド
- ヘッダーに「日報一覧」タイトル + 新規作成ボタン

#### カードデザイン
- 日付（太字）+ できごと冒頭50文字
- よかったこと有り → ✨アイコン
- 改善点有り → 📝アイコン
- タップで詳細へ遷移

#### ページング
- 無限スクロール（30件ずつ追加読込）
- ローディングはスピナー表示

### 詳細画面

#### レイアウト
- ヘッダー：戻るボタン + 日付
- コンテンツ：セクションごとにカード化
- フッター：編集ボタン（Phase 1では任意）

#### セクション表示
- 「できごと」「学び」「よかったこと」「改善点」を見出し付きで表示
- 空の項目は「（なし）」表示 or 非表示
