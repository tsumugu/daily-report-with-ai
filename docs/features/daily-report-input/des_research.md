# 日報入力機能 デザインリサーチ（Des Research）

## 1. PRD要求サマリ
- 日報入力（日付、できごと、学び、よかったこと、改善点/アクション）
- よかったこと・改善点は「要因」「次のアクション」とセットで記録可能
- 保存完了メッセージ or 一覧画面へ遷移
- モバイル/PC両対応

---

## 2. 一般的なUI/UXパターン

### 2.1 フォーム設計のベストプラクティス

| 要素 | ベストプラクティス |
|:-----|:------------------|
| **レイアウト** | 1カラム、上から下へ自然な流れ |
| **ラベル** | フィールド上部配置、必須マーク明示 |
| **Textarea** | 自動リサイズ or 十分な初期高さ |
| **日付選択** | DatePicker、デフォルト当日 |
| **保存ボタン** | 画面下部固定（モバイル）、目立つ色 |

### 2.2 長文入力のUX

| パターン | 特徴 |
|:---------|:-----|
| **プレースホルダー例文** | 何を書くべきかガイド |
| **文字数カウンター** | 上限がある場合に表示 |
| **自動保存（下書き）** | 入力途中でも安心（Phase 2） |
| **セクション分け** | 項目ごとにカード化 |

### 2.3 複数項目追加UI

| パターン | 特徴 | 適用ケース |
|:---------|:-----|:-----------|
| **＋追加ボタン** | 項目を動的に追加 | よかったこと複数入力 |
| **アコーディオン** | 折りたたみで省スペース | 項目が多い場合 |
| **インラインフォーム** | その場で追加 | シンプルな追加 |

## 4. 推奨デザイン方針（本プロジェクト向け）

### レイアウト
- モバイルファースト、縦1カラム
- セクションごとにカード化（できごと / 学び / よかったこと / 改善点）
- よかったこと・改善点は「＋追加」ボタンで複数入力可能

### 入力体験
- 日付はDatePicker、デフォルト当日
- Textareaはプレースホルダーで記入例表示
- 文字数カウンター（上限1000文字）
- 必須項目は赤いアスタリスク（*）

### 保存体験
- 保存ボタンは画面下部に固定（モバイル）
- 保存中はローディングスピナー
- 保存完了はトースト通知 → 一覧へ自動遷移

### エラー・バリデーション
- リアルタイムバリデーション（blur時）
- エラーはフィールド直下に赤字表示
- 未入力で保存試行 → 該当フィールドにスクロール
