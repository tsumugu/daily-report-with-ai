# 日報入力機能 UI/UX設計書（UI Design）

## 1. 概要
日報入力画面のUI/UX設計方針を定義する。

## 2. 画面構成

### 日報入力画面
- ヘッダー：画面タイトル「日報を書く」、戻るボタン
- フォーム部：各入力フィールド
- フッター：保存ボタン

## 3. UIコンポーネント

| 項目 | コンポーネント | 備考 |
|:-----|:--------------|:-----|
| 日付 | DatePicker | デフォルト: 当日 |
| できごと | Textarea | 必須、プレースホルダーで記入例表示、文字数カウンター |
| 学び | Textarea | 任意、文字数カウンター |
| よかったこと | カード + Textarea×2 | 内容 + 要因、複数追加可（＋追加ボタン）※タグ入力はPhase 2 |
| 改善点/アクション | カード + Textarea×2 | 内容 + アクション、複数追加可（＋追加ボタン） |
| 追加ボタン | Button (Secondary) | 「＋ 追加」 |
| 保存ボタン | Button (Primary) | 「保存する」、モバイルでは下部固定 |

## 4. レイアウト方針
- モバイルファースト設計
- 縦1カラム配置、スクロールで全項目表示
- PC時は中央寄せ・最大幅制限で見やすく

## 5. インタラクション
- 必須項目未入力時は保存ボタン非活性 or エラー表示
- 保存完了時はトースト通知 + 一覧画面へ遷移
- 入力中に離脱時は確認ダイアログ（任意実装）

## 6. デザイントークン
- Shadow-CN / Tailwind CSSベース
- カラー・フォント・余白はデザインシステム準拠

## 8. 状態別表示

### ローディング状態
- 保存ボタン押下後：ボタン非活性 + スピナー表示

### エラー状態
- バリデーションエラー：該当フィールド下に赤字メッセージ
- 必須未入力：フィールド枠を赤色に

### 成功状態
- 保存完了：トースト通知「日報を保存しました」→ 一覧へ遷移
