# 日報入力機能 要件定義書（PRD）

## 1. 概要
ユーザーが毎日の業務後に「日報」を入力・保存できる機能。  
できごと・学び・よかったこと・改善点/アクションを記録し、振り返りの土台を作る。

## 2. ユーザーストーリー
- 新卒社員として、業務終了後にスマホ or PCで日報を簡単に入力したい
- よかったことや改善点を「要因」「次のアクション」とセットで記録したい
- 入力完了後、保存されたことを確認できる

## 3. 機能要件

### 3.1 入力項目
| 項目名 | 必須/任意 | 説明 |
|:-------|:---------|:-----|
| 日付 | 必須 | 対象の業務日（デフォルト: 当日） |
| できごと | 必須 | その日あった主な出来事・業務内容 |
| 学び | 任意 | 業務を通じて得た気づき・学び |
| よかったこと | 任意 | うまくいった点、成功体験（要因も入力可） |
| 改善点/アクション | 任意 | 改善したいこと、明日以降に試したいこと |

### 3.2 操作フロー
1. 日報入力画面を開く
2. 各項目を入力
3. 「保存」ボタンで日報を登録
4. 保存完了メッセージ表示 or 一覧画面へ遷移

### 3.3 バリデーション
- 日付・できごとは必須
- 文字数上限（各項目1000文字程度を想定）

## 4. 非機能要件
- モバイル/PC両対応（レスポンシブ）
- 入力途中の下書き保存（Phase 2以降で検討）

## 5. 成功基準
- ユーザーが毎日日報を書いて保存できる
- 入力〜保存までの体験がストレスなく完了する
