# ユーザー認証機能 デザインリサーチ（Des Research）

## 1. PRD要求サマリ
- ログイン画面（メール/パスワード入力、ログインボタン）
- サインアップ画面（メール/パスワード/確認入力、登録ボタン）
- ログアウト（ヘッダー等から実行、ログイン画面へ遷移）
- モバイル/PC両対応

---

## 2. 一般的なUI/UXパターン

### 2.1 ログイン画面のベストプラクティス

| 要素 | ベストプラクティス |
|:-----|:------------------|
| **レイアウト** | 中央配置、余白を十分に取る |
| **フォーム** | 1カラム、ラベルはフィールド上部 |
| **パスワード** | 表示/非表示トグル必須 |
| **エラー表示** | フィールド直下に赤字、具体的なメッセージ |
| **CTA** | 「ログイン」ボタンは目立つ色、幅広 |
| **補助リンク** | 「パスワードを忘れた」「アカウント作成」 |

### 2.2 サインアップ画面のベストプラクティス

| 要素 | ベストプラクティス |
|:-----|:------------------|
| **項目数** | 最小限（メール、パスワード、確認のみ） |
| **パスワード強度** | リアルタイムで強度表示（任意） |
| **確認入力** | パスワード確認は必須 |
| **利用規約** | チェックボックス + リンク |
| **CTA** | 「アカウントを作成」等、行動を明示 |

### 2.3 共通UXポイント

- **ローディング状態**：ボタン押下後はスピナー表示
- **エラー時**：フィールドを赤枠 + メッセージ
- **成功時**：トースト通知 or 自動遷移
- **キーボード操作**：Enter送信対応、Tab順序

## 4. 推奨デザイン方針（本プロジェクト向け）

### レイアウト
- モバイルファースト、中央寄せ1カラム
- ロゴ/サービス名を上部に配置
- 余白を十分に取り、入力しやすさ重視

### カラー
- CTA（ログイン/登録ボタン）はプライマリカラー
- エラーは赤系、成功は緑系
- 背景はライト（ダークモードはPhase 2以降）

### インタラクション
- パスワード表示トグル必須
- バリデーションはリアルタイム（blur時）
- ローディング中はボタン非活性 + スピナー

### アクセシビリティ
- ラベルとinputの紐付け（for属性）
- フォーカス状態の視認性確保
- エラーメッセージはaria-live対応
