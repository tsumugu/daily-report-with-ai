# ユーザー認証機能 要件定義書（PRD）

## 1. 概要
ユーザーがサービスにログイン/ログアウトできる最低限の認証機能。  
日報データは本人のみがアクセスできるよう、認証によるアクセス制御を行う。

## 2. ユーザーストーリー
- 新卒社員として、自分専用のアカウントでログインしたい
- 自分の日報は他人に見られたくない
- ログアウトして安全にセッションを終了したい

## 3. 機能要件

### 3.1 認証方式（MVP）
- メールアドレス + パスワードによるログイン
- 新規登録（サインアップ）機能
- ログアウト機能

### 3.2 画面・操作フロー
| 画面 | 説明 |
|:-----|:-----|
| ログイン画面 | メール/パスワード入力、ログインボタン |
| サインアップ画面 | メール/パスワード/確認入力、登録ボタン |
| ログアウト | ヘッダー等から実行、ログイン画面へ遷移 |

### 3.3 バリデーション
- メール形式チェック
- パスワード最低文字数（8文字以上推奨）
- 登録済みメールの重複チェック

### 3.4 セキュリティ
- パスワードはハッシュ化して保存
- セッション管理（トークンベース or Cookie）
- 未認証時は日報機能へアクセス不可

## 4. 非機能要件
- モバイル/PC両対応
- ソーシャルログイン（Google等）はPhase 2以降で検討

## 5. 成功基準
- ユーザーが自分のアカウントで安全にログイン/ログアウトできる
- 他人の日報データにはアクセスできない
