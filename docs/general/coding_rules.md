# 💻 コーディングルール（全体共通）

## 1. Lint（静的解析）
- コード品質と一貫性を保つため、**ESLint**によるルールチェックを必須とします。
- ルール本体は `src/` 配下の設定ファイル（.eslintrc等）で一元管理します。
- Lintのルール例：命名規則, unused importの禁止, 自動フォーマット推奨 …etc

## 2. コミット前チェック
- **コミット前に必ず下記2点を実行すること**：
  1. `yarn lint` または `npm run lint`
  2. `yarn test` または `npm run test`
- Lint もしくは Test でエラーが出た場合は必ず修正してからコミットしてください。

## 3. その他
- 詳細な個別ルールは `src/` 配下のESLint設定およびチーム合意のrosに準じます。
- 質問や不明点は気軽にレビュワー・メンバーまで。
