## Phase 2 v2：フォローアップ機能拡張

### Phase 2 v2-1：フォローアップ動作改善（最優先）

**ゴール**

- エピソード/アクションを複数登録できるようにし、ステータスを自動判定することで、フォローアップ記録の負担を軽減する

- 登録数に基づいてステータスを自動判定（0件=未着手、1件以上=進行中、閾値以上=再現成功/完了）
- 閾値: 3回（初期値、固定）

**成功基準**

- ユーザーがエピソード/アクションを複数登録できる
- 登録数に応じてステータスが自動的に更新される
- 既存のフォローアップ機能との互換性が保たれる

**ステータス**: 📋 設計中（PRD作成済み、tech_spec/ui_design未着手）

## Phase 3：編集機能と中長期目標連携

**ゴール**

- 入力した日報やフォローアップを後から編集できるようにし、記録の正確性を向上させる
- 中長期目標と日々の振り返りを接続し、目標達成への道筋を可視化する

**主な機能**

### Phase 3-1：編集機能

- **日報の編集**
  - 日報詳細画面から編集可能
  - 入力項目（できごと・学び・よかったこと・改善点/アクション）を編集
  - 編集履歴の保持（Phase 3以降で検討）

- **フォローアップのエピソードとアクションの編集**
  - エピソード/アクションの日付・メモを編集可能
  - フォローアップ画面から編集ボタンを配置
  - 編集後、ステータスを自動再計算

**成功基準**

- ユーザーが誤入力や追記を簡単に修正できる
- 編集操作が直感的で、ストレスなく完了できる

**ステータス**: 📋 未着手

**実装優先度**: Phase 3の最初の機能

---

### Phase 3-2：中長期目標との接続機能

**ゴール**

- 長期目標から短期目標、そして週次フォーカスまで、目標の階層構造を構築し、日々の振り返りと目標達成を接続する

**階層構造**

```
長期目標（例: 半期目標）
  └── 中期目標（長期目標を達成するための目標）
        └── 短期目標（1週間スパン、中期目標を達成するための目標）
              └── 週次フォーカス（短期目標のためのアクションとして）
```

**主な機能**

- **長期目標の管理**
  - 長期目標の作成・編集・削除
  - 目標期間の設定（例: 半期、四半期）
  - 目標の説明・達成基準の設定

- **中期目標の管理**
  - 長期目標に紐づく中期目標の作成・編集・削除
  - 中期目標の期間設定（例: 1ヶ月、2ヶ月）
  - 長期目標への紐付け

- **短期目標の管理**
  - 中期目標に紐づく短期目標の作成・編集・削除
  - 短期目標の期間設定（1週間スパン）
  - 中期目標への紐付け
  - 週次フォーカスとの接続

- **週次フォーカスとの接続**
  - 短期目標から週次フォーカスを設定可能
  - 週次フォーカスから短期目標を参照可能
  - 目標達成状況の可視化

- **目標達成状況の可視化**
  - 長期目標 → 中期目標 → 短期目標 → 週次フォーカスの進捗を階層的に表示
  - 目標達成率の計算・表示
  - 目標と日報の紐付け（日報から目標への関連付け）

**成功基準**

- ユーザーが目標の階層構造を理解し、設定できる
- 日々の振り返り（日報・フォローアップ）と目標が接続され、達成への道筋が明確になる
- 目標達成率が可視化され、モチベーションが向上する

**ステータス**: 📋 未着手

**実装優先度**: Phase 3-1完了後

## あるべき姿の議論

**方針**

Phase 3以降の機能は一旦白紙とし、以下の観点からあるべき姿を議論していく：

### 議論の観点

1. **ユーザー価値**
   - どの機能がユーザーにとって最も価値が高いか
   - ユーザーフィードバックから見えるニーズは何か

2. **市場ニーズ**
   - 競合サービスとの差別化ポイントは何か
   - 市場で求められている機能は何か

3. **技術的実現可能性**
   - 実装コストと効果のバランス
   - 技術的制約やリスク

4. **プロダクトビジョン**
   - このプロダクトが目指すべき方向性
   - 長期的な価値提供のあり方

### 検討中の機能アイデア（参考）

- **分析・可視化機能**
  - 日報データの傾向分析
  - 成長グラフ・統計情報
  - 半期目標との連携

- **検索・フィルタリング機能**
  - 高度な検索機能
  - タグ・カテゴリでの分類
  - データエクスポート

- **コラボレーション機能**
  - メンター/上司によるフィードバック
  - チームでの日報共有
  - 組織展開対応

- **AI機能**
  - 日報の自動サマリー生成
  - 成功/失敗の自動判定
  - 傾向分析の自動化

**次のステップ**

1. ユーザーフィードバックの収集・分析
2. 市場調査・競合分析
3. プロダクトビジョンの再確認
4. 優先順位の議論と決定
