## Phase 1：MVP（最小限の価値検証）

**ゴール**

- 日報を「できごと・学び・よかったこと・改善点」の形式で入力・保存できる
- 入力した日報を一覧・詳細で確認できる

**主な機能**

- 日報入力画面（できごと/学び/よかったこと/改善点/アクション）
- 日報一覧・詳細表示
- モバイル/PC対応のレスポンシブUI
- ユーザー認証（最低限のログイン機能）

**成功基準**

- ユーザーが毎日日報を書いて保存できる
- 過去の日報を一覧・参照できる

## Phase 2：フォローアップ・再現性向上

**ゴール**

- 「よかったこと」「改善アクション」を後追い管理し、書きっぱなしを防ぐ

**主な機能**

- よかったこと／改善事項へのタグ・要因追加
- 「再現できたか」「アクション進捗」のフォローアップ入力
- リマインダー通知（プッシュ/メール等）
- フォロー項目一覧・ステータス管理

**成功基準**

- よかったこと再現率・改善アクション実施率が可視化される
- ユーザーがリマインドを通じて振り返り行動を取れる

**ステータス**: ✅ 完了（followup-management, weekly-focus-management実装済み）

## Phase 2 v2：フォローアップ機能拡張

### Phase 2 v2-1：フォローアップ動作改善（最優先）

**ゴール**

- エピソード/アクションを複数登録できるようにし、ステータスを自動判定することで、フォローアップ記録の負担を軽減する

**主な機能**

- よかったことの再現エピソードを複数登録できる
- 改善点のアクションを複数登録できる
- 登録数に基づいてステータスを自動判定（0件=未着手、1件以上=進行中、閾値以上=再現成功/完了）
- 閾値: 3回（初期値、固定）

**成功基準**

- ユーザーがエピソード/アクションを複数登録できる
- 登録数に応じてステータスが自動的に更新される
- 既存のフォローアップ機能との互換性が保たれる

**ステータス**: 📋 設計中（PRD作成済み、tech_spec/ui_design未着手）

**関連ドキュメント**

- [フォローアップ動作改善 PRD](./features/followup-enhancement/prd.md)

**実装優先度**: 最優先（Phase 2 v2の最初の機能）

## あるべき姿の議論

**方針**

Phase 3以降の機能は一旦白紙とし、以下の観点からあるべき姿を議論していく：

### 議論の観点

1. **ユーザー価値**
   - どの機能がユーザーにとって最も価値が高いか
   - ユーザーフィードバックから見えるニーズは何か

2. **市場ニーズ**
   - 競合サービスとの差別化ポイントは何か
   - 市場で求められている機能は何か

3. **技術的実現可能性**
   - 実装コストと効果のバランス
   - 技術的制約やリスク

4. **プロダクトビジョン**
   - このプロダクトが目指すべき方向性
   - 長期的な価値提供のあり方

### 検討中の機能アイデア（参考）

- **分析・可視化機能**
  - 日報データの傾向分析
  - 成長グラフ・統計情報
  - 半期目標との連携

- **検索・フィルタリング機能**
  - 高度な検索機能
  - タグ・カテゴリでの分類
  - データエクスポート

- **コラボレーション機能**
  - メンター/上司によるフィードバック
  - チームでの日報共有
  - 組織展開対応

- **AI機能**
  - 日報の自動サマリー生成
  - 成功/失敗の自動判定
  - 傾向分析の自動化

**次のステップ**

1. ユーザーフィードバックの収集・分析
2. 市場調査・競合分析
3. プロダクトビジョンの再確認
4. 優先順位の議論と決定
