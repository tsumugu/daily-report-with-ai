# バックログ

**目的**: すぐに開発する予定のアイテムを管理する

**管理方法**: PdM が Discovery の成果物で Adopt と判断したものを Queue する

**更新頻度**: Phase 5（棚卸し）のたびに更新

---

## フォーマット

### アイテムの記録形式

```markdown
## BL-{番号}: {アイテム名}

**優先順位**: P0 / P1 / P2 / P3  
**ステータス**: Queue / In Progress / Done  
**Queue日**: YYYY-MM-DD  
**判断理由**: [Adoptと判断した理由]  
**関連Discovery**: `.discovery/handoff.md` (YYYY-MM-DD)  
**関連問題**: P-{xx} (problem_statements.md)  
**関連要求**: R-{xx} (requirements.md)

### 概要

[アイテムの概要]

### 次のアクション

- [ ] Phase 1: 設計開始
- [ ] PRD作成
- [ ] その他

### 備考

[その他のメモ]
```

### 優先順位

- **P0**: 最優先（すぐに開発を開始）
- **P1**: 高優先度（次の開発候補）
- **P2**: 中優先度（順次開発）
- **P3**: 低優先度（余裕があれば開発）

### ステータス

- **Queue**: バックログにQueueされている（開発未着手）
- **In Progress**: 開発中（Phase 1〜4のいずれか）
- **Done**: 開発完了（Phase 4完了）

### アイテムの追加方法

1. Discovery の成果物で **Adopt** と判断されたアイテムを追加
2. 優先順位（P0〜P3）を決定
3. ステータスを **Queue** に設定
4. 関連情報（Discovery、問題、要求）を記録

### ステータスの更新

- **Queue → In Progress**: Phase 1（設計）開始時に更新
- **In Progress → Done**: Phase 4（完了）完了時に更新

## BL-01: 目標階層管理機能

**優先順位**: P1  
**ステータス**: In Progress  
**Queue日**: 2025-12-13  
**判断理由**: Discoveryの結果、目標の階層構造を定義・管理する機能は他の機能の前提となる基盤機能として最優先で開発すべきと判断。既存ロードマップの「Phase 3-2：中長期目標との接続機能」と整合性があり、優先度を上げて検討する価値がある。  
**関連Discovery**: `.discovery/handoff.md` (2025-12-13)  
**関連問題**: P-01 (problem_statements.md)  
**関連要求**: R-01 (requirements.md)  
**既存ロードマップ**: Phase 3-2（中長期目標との接続機能）を統合

### 概要

目標を階層的に定義・管理する機能。長期目標から中期目標、短期目標へと階層的にブレイクダウンできる。目標間の関係性（因果関係、依存関係）を明確に可視化し、目標の性質に応じたブレイクダウン方法をサポートする。

**主な機能**:

- 目標を階層的に定義・管理（階層数は柔軟に設定可能、基本は2〜3階層）
- 目標の期間設定は柔軟に対応（半期、四半期、1ヶ月、1週間など、目標の性質に応じて設定可能）
- 目標間の階層構造を一元的に管理
- 目標間の関係性（因果関係、依存関係）を明確に可視化
- 目標の性質に応じたブレイクダウン方法をサポート（例: スキル習得、プロジェクト完了、習慣形成など）
- 既存の週次フォーカス機能との接続

## BL-02: 目標の落とし込み（ブレイクダウン）サポート機能

**優先順位**: P1  
**ステータス**: Queue  
**Queue日**: 2025-12-13  
**判断理由**: 目標の階層構造（BL-01）が実現された後、目標の落とし込みをサポートする仕組みにより、ユーザーが目標を階層的に設定しやすくなる。目標設定のハードルを下げ、より多くのユーザーが目標を階層的に設定できるようになる。  
**関連Discovery**: `.discovery/handoff.md` (2025-12-13)  
**関連問題**: P-05 (problem_statements.md)  
**関連要求**: R-05 (requirements.md)

### 概要

目標の落とし込み（ブレイクダウン）をサポートする仕組み。テンプレート、ガイド、推奨事項などを提供し、目標設定の質を向上させる（SMART基準のチェック機能など）。

**主な機能**:

- 目標の落とし込み（ブレイクダウン）をサポートする仕組み（例: テンプレート、ガイド、推奨事項など）
- 目標設定の質を向上させる仕組み（例: SMART基準のチェック機能）
- 目標の性質に応じたブレイクダウン方法のガイド
- 目標設定のベストプラクティスの共有

### 次のアクション

- [ ] PRD作成

## BL-03: 日報と目標の関連付け機能

**優先順位**: P1  
**ステータス**: Queue  
**Queue日**: 2025-12-13  
**判断理由**: 日々の活動と目標を接続することで、目標達成への道筋が明確になる。ユーザー価値が高い。  
**関連Discovery**: `.discovery/handoff.md` (2025-12-13)  
**関連問題**: P-02 (problem_statements.md)  
**関連要求**: R-02 (requirements.md)

### 概要

日報の内容（やったこと、よかったこと、改善点）と目標を関連付ける機能。関連付けの負担を軽減する仕組み（タグ付け、自動関連付けなど）を提供する。

**主な機能**:

- 日報の内容と目標を関連付けることができる
- 関連付けの負担を軽減する仕組み（例: タグ付け、自動関連付けの検討など）
- 日々の活動がどの目標の達成に貢献しているかを明確にできる
- 目標達成のための具体的なアクションが日報から抽出できる
- 日報と目標の関連性を可視化できる
- 複数の目標に関連付けられた場合、目標ごとの貢献度を定量化できる（任意）

### 次のアクション

- [ ] Phase 1: 設計開始（BL-01完了後）
- [ ] PRD作成

### 備考

- BL-01（目標階層管理機能）の実現後に開発を開始
- 関連付けの負担を軽減する仕組みを優先的に検討すること
- 日報と目標の関連付けは任意（必須ではない）

## BL-04: フォローアップアクションと目標階層の接続機能

**優先順位**: P1  
**ステータス**: Queue  
**Queue日**: 2025-12-13  
**判断理由**: 既存のフォローアップ機能との接続により、既存機能の価値を高めることができる。短期・中期・長期すべての目標階層との関係を可視化することで、より包括的な目標達成の可視化が可能になる。  
**関連Discovery**: `.discovery/handoff.md` (2025-12-13)  
**関連問題**: P-03 (problem_statements.md)  
**関連要求**: R-03 (requirements.md)

### 概要

フォローアップアクションと短期・中期・長期目標を接続する機能。各目標階層の達成度や進捗状況を可視化する。

**主な機能**:

- フォローアップアクションと短期・中期・長期目標を関連付けることができる
- フォローアップアクションが短期・中期・長期目標の達成にどう貢献しているかを明確にできる
- 各目標階層（短期・中期・長期）の達成度や進捗状況を可視化できる
- 目標達成への道筋が明確になる

### 次のアクション

- [ ] PRD作成

### 備考

- BL-01（目標階層管理機能）の実現後に開発を開始
- 既存のフォローアップ機能との互換性を保つこと
- フォローアップアクションと目標の関連付けは任意（必須ではない）

## BL-05: 目標達成の進捗状況可視化機能

**優先順位**: P2  
**ステータス**: Queue  
**Queue日**: 2025-12-13  
**判断理由**: 目標達成のモチベーション維持に貢献するが、他の機能（BL-01, BL-02, BL-03, BL-04）が実現された後の方が効果的。  
**関連Discovery**: `.discovery/handoff.md` (2025-12-13)  
**関連問題**: P-04 (problem_statements.md)  
**関連要求**: R-04 (requirements.md)

### 概要

目標達成の進捗状況を可視化する機能。目標の性質に応じた進捗率の計算方法を定義し、階層的に進捗を把握できる。

**主な機能**:

- 長期目標、中期目標、短期目標それぞれの達成状況を可視化できる
- 目標達成率を計算・表示できる（目標の性質に応じた計算方法を定義）
- 目標達成に向けた進捗を階層的に把握できる
- 目標達成の進捗状況を一元的に確認できる（ダッシュボード等）
- 下位目標の進捗を上位目標に集約する方法が明確であること（例: 重み付け平均、完了率など）

### 次のアクション

- [ ] Phase 1: 設計開始（BL-01, BL-02, BL-03, BL-04完了後）
- [ ] PRD作成

### 備考

- BL-01, BL-02, BL-03, BL-04の実現後に開発を開始
- 目標達成率の計算方法は明確に定義すること（目標の性質に応じた計算方法を検討）
- 進捗の集約方法（重み付け平均、完了率など）を明確に定義すること
